Redis 穿透 击穿 雪崩

更新于 2020-11-28

缓存穿透

（恶意）访问大量在 MySQL 中不存在的记录，由于记录不存在、不会写 Redis 缓存，每次请求实际上都会打到 MySQL 上，效果上看起来就像是穿透/绕过（bypass）了缓存保护。

解决方式：

  • 布隆过滤器
  • 空对象

缓存击穿

十分热点的 key 过期，大量的进程/线程同时访问数据库。

解决方式：

  • 设置热点 key 永不过期
  • 分布式锁，保证任意时刻 1-3 中只有一个可以获得锁从而访问到数据库

缓存雪崩

key 集中过期，及 redis 宕机。

解决方式：

  • 数据预热，预先访问/加载数据，设置不同的过期时间，让缓存过期时间点分布尽可能均匀
  • 集群，异地多活
  • 限流，服务质量降级，通过锁或者队列控制访问到数据库的线程/进程数目
