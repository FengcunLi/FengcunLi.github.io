std::set_new_handler

std::new_handler set_new_handler( std::new_handler new_p ) noexcept;
Makes new_p the new global new-handler function and returns the previously installed new-handler.

The new-handler function is the function called by allocation functions whenever a memory allocation attempt fails. Its intended purpose is one of three things:

  1. make more memory available
  2. terminate the program (e.g. by calling std::terminate)
  3. throw exception of type std::bad_alloc or derived from std::bad_alloc

At program startup, new-handler is a null pointer.

new/array new expression , delete/array delete expression

new 表达式就做了两件事：

  • 调用 operator new allocation function 分配空间
  • 调用构造函数

delete 表达式就做了两件事：

  • 调用析构函数
  • 调用 operator delete deallocation function 释放空间

array new 表达式就做了两件事：

  • 调用 operator new[] allocation function 分配空间
  • 调用默认构造函数(只能使用无参数构造函数)

array delete 表达式就做了两件事：

  • 调用析构函数
  • 调用 operator delete[] deallocation function 释放空间

placement new 表达式就做了两件事：

  • 调用 operator new(size_t, void*)
  • 调用构造函数

operator new, operator delete, operator new[], operator delete[], operator new(size_t, void*) 都可以被重载，从而接管内存管理。

表达式 new, new[], delete,delete[],new(...) 是不可以被重载的。

数组 vs 指针

杜绝野指针

野指针不是 NULL 指针，是指向“垃圾”内存的指针， 指针 p 被 free 或者 delete 之后，没有置为 NULL，让人误以为 p 是个合法的指针。

  • 函数的 return 注意不要返回指向“栈内存”的“指针”或者“引用”
  • free 或 delete 释放内存后，将指针设置为 NULL，避免产生“野指针”
  • 用 malloc 申请内存后，应该立即检查指针值是否为 NULL，避免使用值为 NULL 的指针

禁止产生堆对象

通过将 allocation function 和 deallocation function 声明为私有，可以阻止 new 表达式和 delete 表达式从外部对它们的访问，禁止 new 表达式和 delete 表达式的正确执行，禁止堆对象的产生。

禁止产生栈对象

将类的构造函数和析构函数声明为私有或者受保护，这样就不能直接通过外部代码实例化一个栈对象了，但同时也禁止了 new 表达式和 delete 表达式的正常执行，解决这一个问题的方式是将 new 表达式和 delete 表达式放到类的静态方法和实例方法中，这样 new 和 delete 就可以访问到类的私有或者受保护的构造函数和析构函数了。
