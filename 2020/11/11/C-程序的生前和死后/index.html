C++ 程序的生前和死后

更新于 2020-11-11

The /ENTRY option specifies an entry point function as the starting address for an .exe file or DLL.

The function must be defined to use the __stdcall calling convention. The parameters and return value depend on if the program is a console application, a windows application or a DLL. It is recommended that you let the linker set the entry point so that the C run-time library is initialized correctly, and C++ constructors for static objects are executed.

By default, the starting address is a function name from the C run-time library. The linker selects it according to the attributes of the program, as shown in the following table.

双向循环链表：

  • task list: 进程描述符（task struct 1.7KB?）list。内存地址空间数十兆。
  • malloc: small block heap 的每一个 block 也是使用双向循环链表串起来的。切小了，换一个链表；当发生区块的释放时，虽然逻辑上不相邻（散落在不同的链表中），但是由于“物理相邻”，可以发生上下合并，从而再换一个大一点的链表。
  • 在 Debug 模式下，每一块分配出去的内存通过双向循环链表串起来形成“Debug Heap”，也就是说每一块分配出去的内存，都握在调试器手里，这也是为什么调试器可以做那么多神奇的事情。

单向链表：

  • GNU C std::alloc 16 个单向链表，分别挂着 8 至 128 字节的块。插入和取出的时间复杂度为 O(1)，这也是为什么此处选择这一数据结构。

内存池的核心思想就是挖一大块，然后来做切割，并将相同大小的区块串在一块，这些相同大小的区块是在运行过程中不断产生的，与其让它们散落地分布，不如就将他们分组地串起来，让相同大小的区块串在一起。下次再要一个区块时，直接去对应的链表拿就好了。
