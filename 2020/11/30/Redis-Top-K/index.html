Redis Top-K

更新于 2021-02-13

排行榜，比如粉丝排行榜，播放排行榜等，排行榜本身含义非常清晰，就是维护一个列表，其中每个条目有一个 score，依据 score 进行排序，形成排行榜。

redis zset 由于如下特性天生支持排行榜需求：

  • zset 是有序集合类型
  • zset 支持常用的添加，删除，范围读取操作
  • 有序集合的成员是唯一的，但分数(score)却可以重复

redis zset 实现排行榜功能

将用户 id 作为 member，用户粉丝量作为 score

  1. zset 添加 member
    如果用户粉丝数目发生变动，则需要将该用户重新添加到 zset 中进行排序，假定用户 id 为 user_id，粉丝量为 score

  2. zset 维持 Top-K

    范围是 0 到 -(K+1)，因为 redis zset 是按照从小到大方式排序的，所以需要维持的榜单是倒数 K，也即从最后一个元素开始，倒数推 K 个

    复杂度：

  3. zset 删除 member
    如果某个用户进行注销操作，或者被封号了，这个时候该用户应该从 redis zset 中被踢掉

  4. 最后是获取排行榜 Top-K 数据

    • 这里范围是 0 到 K-1，原因是：redis zset 下标从 0 开始；
    • ZREVRANGE 是倒序取数据；
    • withscores 会将 member 与对应 score 一起返回
