Iterator Invalidation Rules

更新于 2021-03-03

Iterator invalidation rules

unordered_map

指针和引用

Rehash 发生时，会重新创建 buckets array，但是 key 和 value 不会拷贝也不会析构，仅仅是指针的指向改一改，重新链接一下，不会引起指向 key 和 value 的指针和引用失效。
指向 key 和 value 的指针和引用只有在该 key 被 erase（delete，调用析构函数）时才会失效。

迭代器

Rehash 发生时，由于发生了重新链接，会导致所有的迭代器全部失效，迭代器的加加减减移动操作不再具有意义，迭代器的核心就是迭代移动，当其所指之处的上下文发生了变化时，这个迭代器也就是失效了；
erase 时，仅仅是指向 key 所指节点的迭代器会失效，其他的迭代器依然可以快快乐乐地加加减减解引用。

map

erase 时，指针、引用、迭代器 to the erased elements 会失效，其他的指针、引用、迭代器依旧有效，特别是迭代器依然可以快快乐乐地加加减减解引用，在树的结构上完成遍历。
insert 时， No iterators or references are invalidated.

list

在 list 内或者 list 间移动元素，不会引起指针、引用、迭代器的失效，迭代器依然可以快快乐乐地加加减减解引用，在链上移动。

在 list 中删除元素（们），仅仅被删除元素的指针、引用、迭代器会失效，其他的指针、引用、迭代器依旧有效。

vector

  • 有新对象的生成，就有构造函数的调用
  • 新位置拷贝/移动构造，原位置析构释放
