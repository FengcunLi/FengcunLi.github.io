Shards

Slack is a very successful modern service that’s built with shards.不同的 Slack organization 被 shard 到不同的 shard 中，不同的 shard 彼此同质且独立。

这个方案不利于数据分析，因为数据是分散的，解决方案如下，aggregate into one large database for analysis and logging.

Lambda



Lambda makes distinction between Streaming and Batch (Unbounded VS Bounded).
流的概念第一次出现。

Kafka is usually involved in lambda architecture.

Streaming


解决的事就是 OLTP 和 OLAP（data warehouse，DWH） 之间的 Gap，以及 OLTP 中各个微服务之间的 Gap，方案就是 Kafka as Bus and CDC。

KEY: 使用 Kafka 作为 message bus.

Kafka 对待流处理的方式是，keep stream computation very close to where the application code is. 这一点和 spark stream 不一样，Kafka’s approach to stream 不需要额外的 engine 来进行流处理，而是使用 application code 实现流处理逻辑。

The retention policy is configurable. The retention policy in Kafka is 7 days by default. The New York Times stores all of its newspaper data going back to the 1860s in Kafka topics.
