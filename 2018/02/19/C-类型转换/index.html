static_cast

static means compile-time.

隐式类型转换

编译器进行 implicit conversion 的地方，就该用 static_cast

  • float to/from int
  • char to/from int

Easy to find where conversion is happening, 出于最佳实践的考虑。

调用 conversion constructor 及 conversion operator 的地方，相较于 implicit conversion，就该用 static_cast

危险类型转换

static_cast is more restrictive than C-Style
char * to int * is allowed in C-Style but not with static_cast

static_cast 可以帮助我们察觉并避免很多危险的类型转换，出于检查、规避危险的考虑。

dynamic_cast

指针或者引用向下类型转换
将基类指针类型转换为派生类指针，并检查目标指针类型与该基类指针实际指向的对象的类型是否匹配。

dynamic_cast<new-type>(expression)

  • new-type - pointer to complete class type, reference to complete class type, or pointer to (optionally cv-qualified) void
  1. 基类至少有一个虚函数，使之成为多态类。
  2. If the cast is successful, dynamic_cast returns a value of type new-type.
  3. If the cast fails and new-type is a pointer type, it returns a null pointer of that type.
  4. If the cast fails and new-type is a reference type, it throws an exception that matches a handler of type std::bad_cast.

BOTTOM LINE

  1. work only on polymorphic base class (at least one virtual function in base class) because it uses this information to decide about wrong down-cast.
  2. it is used for up-cast (D to B) and down-cast (B to D), but it is mainly used for correct downcast.
  3. using this cast has run time overhead, because it checks object types at run time using RTTI (Run Time Type Information).
  4. if we are sure that we will never cast to wrong pointer type then we should always avoid this cast and use static_cast.

const_cast

去掉指针或者引用的 constness。

const_cast<new_type>(expression)

  • expression: pointer or reference
  • new_type: must be a pointer, reference, or pointer-to-member type.

When we need to call some 3’rd party library where it’s taking non-const variable/object but not changing that.但是我们手里只有 const variable/object 时，我们不得不利用 const_cast 去掉指针或者引用的 constness，并且祈祷第三方库真的不修改，otherwise it will become undefined behavior。

使用 const_cast 一般都是不得不为之，因为它带有一定的风险。
Use const_cast only when you have to

reinterpret_cast

重新解释指针/地址
reinterpret_cast can perform dangerous conversions because it can typecast any pointer to any other pointer.

最像 C-Style 的、应用在指针上的强制类型转换
