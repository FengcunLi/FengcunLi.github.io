子类父类的同名函数

子类和父类的方法之间就只能是重写，不允许重载，如果你试图重载发生的就只能是隐藏，子类的方法会完全隐藏父类所有的同名函数，即你在子类中定义了一个父类中函数的同名函数，则父类中所有的同名函数对于子类来说均不可用了，一是因为发生重写，二是因为不允许子类和父类的方法之间发生重载，子类中的这个函数会完全隐藏父类所有的同名函数。

为什么一个类有虚函数就必须要有虚析构函数

用一个基类指针指向一个派生类对象时，不管存不存在虚函数，也就是不管是不是多态类，delete 基类指针都会陷入局部销毁的陷阱，当 delete 基类指针时，其结果是未定义的，实际执行时通常发生的是对象的派生类成分未被销毁。 即调用的是基类的析构函数，而派生类的析构函数未能执行，会造成一个诡异的“局部销毁”对象，这会导致资源泄露，数据结构败坏，在调试器上浪费很多的时间。

通过将析构函数定义为虚函数，可以避免局部销毁。

如果一个类没有虚析构函数，不要继承它

基类应该有个虚析构函数，才是像样的、安全的基类。

有的类不是意图当作基类的，有的基类并不用于多态性质，即并非被设计用来“通过基类的接口处理派生类对象”，这样的类就不需要为其提供虚析构函数的，因为它们就不可能陷入“局部销毁”陷阱。
如果一个类有基类企图，就该为其定义虚析构函数，避免局部销毁。
如果一个类没有基类企图，就不该为其定义虚函数，这是出于空间和移植性的考虑

虚函数的语义就是强制派生类对其进行重写，虚函数的语义也就是基类企图语义，就该为其定义虚析构函数

答案：一个类有虚函数就必须要有虚析构函数：
虚函数->基类企图->虚析构函数->避免局部销毁

由于像 string 这些标准库中的类就不是设计用来当作基类的，所有没有虚析构函数，不该尝试对其进行继承操作。

  • 纯虚函数，抽象类
  • 纯虚析构函数，与普通的纯虚函数不一样，基类一定要提供纯虚析构函数的定义；不然的话就会导致链接错误。析构函数的运作方式是：继承链从下而上调用析构函数，编译器会在类 A 的派生类的析构函数中创建一个对 ~A 的调用动作，所以必须要为这个函数提供一份定义，否则链接器就会报错。

析构函数能抛出异常吗？ 绝对不要

C++ 并不禁止析构函数抛出异常，当它并不鼓励你这样做。如果执行过程中一个异常发生并且没有被捕获，会导致栈回退，也就是会导致对象的析构，如果析构函数又抛出异常，就会导致未定义行为。

  1. 析构函数绝对不要抛出异常，如果析构函数调用的函数可能抛出异常，那么析构函数应该捕获这些异常，然后继续析构或者直接结束程序。
  2. 如果需要对那个函数抛出的异常进行处理，那么应该把那个函数分离出析构函数，在一个普通函数中进行调用。

构造函数和析构函数中能调用虚函数吗？ 绝对不要

绝对不要在构造函数和析构函数中调用虚函数，而且确保它们调用的所有函数也都服从这一约束。

假设一个派生类的对象进行析构，首先调用了派生类的析构函数，然后在调用基类的析构函数时，遇到了一个虚函数，这个时候有两种选择：

  • Plan A 是编译器调用这个虚函数的基类版本，即放弃多态，而我们在基类的析构函数里面写一个虚函数，正常的期望值是在派生类析构时调用这个虚函数的派生类版本，编译器这样的选择偏离了我们的期望
  • Plan B 是编译器调用这个虚函数的派生类版本，但是此时对象的派生类部分已经完成析构，派生类的特有成员变量会呈现未定义状态，这个函数调用会导致未定义行为。

实际情况中编译器使用的是 Plan A，在构造和析构期间不要调用虚函数，因为这类调用从来不能下降至派生类，即编译器放弃了多态，这是合理的，因为此时派生类的特有成员变量会呈现未定义状态，不管是因为尚未定义或是已经销毁，一旦下降至派生类并调用函数可能会访问到这些未定义状态的变量。
在 Plan A 放弃多态的情况下，如果虚函数的基类版本不是纯虚函数，即有虚函数的基类版本可调，不会有严重错误发生，但我们依然会困惑于虚函数机制失效，说不准又是“一张通往彻夜调试的直达车票” – Effective C++

编译器为了规避未定义行为，放弃了多态，而我们却会困惑于虚函数机制的失效。

什么时候改变引用计数？

  1. 构造函数中引用计数初始化为 1；
  2. 拷贝构造函数中引用计数加 1；
  3. 拷贝赋值运算符中，左边的对象引用计数减一，右边的对象引用计数加一；
  4. 析构函数中引用计数减一；
  5. 在拷贝赋值运算符和析构函数中，如果减一后为 0，则调用 delete 释放对象；
  6. 作为函数参数
  7. 作为函数返回值
  8. 给 shared_ptr 赋予一个新值
  9. shared_ptr 被销毁，例如一个局部的 shared_ptr 离开其作用域

内存对齐的原则

CPU 的优化原则: 对于 n 个字节的元素(n=2, 4, 8,…) ,它的首地址能被 n 整除,才能获得最好的性能.设计编译器时可以这样做:对于每一个变量,从当前位置向后找到第一个满足这样条件的地址作为首地址.
结构体长度一定是最长元素的整数倍,这样当其放入数组的时候,才不会为遵守 CPU 的优化原则而产生空隙.

内联函数有什么优点？内联函数与宏定义的区别？

内联函数和普通函数相比可以加快程序运行的速度，因为不需要中断调用，在编译的时候内联函数可以直接被镶嵌到目标代码中。
用内联函数完全取代宏：

  1. 内联函数可利用调试器进行调试，而宏定义不可以;
  2. 编译器会对内联函数的参数类型做安全检查或自动类型转换（同普通函数），而宏定
    义则不会；
  3. 内联函数可以访问类的成员变量，宏定义则不能；
  4. 在类中声明同时定义的成员函数，自动转化为内联函数。

inline 是指嵌入代码，就是在调用函数的地方不是跳转，而是把代码直接写到那里去。对于短小的代码来说，inline 可以带来一定的效率提升，而且和 C 时代的宏函数相比，inline 更安全可靠。可是这个是以增加空间消耗为代价的。至于是否需要 inline 函数就需要根据你的实际情况取舍了, 频繁的调用内联函数和宏定义容易造成代码膨胀，消耗更大的内存而造成过多的换页操作。

inline 一般只用于如下情况：

  1. 一个函数不断被重复调用。
  2. 函数只有简单的几行，且函数不包含 for、while、switch 语句等复杂结构, 否则编译器不会将其当作内联函数看待，而是把它决议成为一个静态函数.
    宏不是函数，只是在编译预处理阶段将程序中有关字符串替换成宏体。
    inline 函数是函数，但在编译中不单独产生代码，而是将有关代码嵌入到调用处。
